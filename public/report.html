UnD — Unemployment Data Visualizer

Raport tehnic · Proiect Tehnologii Web · Facultatea de Informatică, UAIC Iași · 2025–2026

Autori: Albert Andreea, Chiuaru Cosmin — Grupa 3A3
Abstract. UnD este o aplicație web pentru vizualizarea datelor publice despre șomajul din România, acoperind 17 luni (ianuarie 2024 – mai 2025). Permite filtrarea după județ, perioadă, mediu urban/rural, grupă de vârstă și nivel de educație, cu grafice interactive și o hartă coroplețică. Exportul este disponibil în CSV, JSON, SVG și PDF. Back-end-ul este implementat în PHP cu SQLite, fără framework-uri.

1. Introducere

Datele privind rata șomajului sunt publicate lunar de ANOFM și disponibile pe data.gov.ro, însă sub o formă dificil de explorat pentru publicul general. Aplicația UnD oferă o interfață web interactivă care nu necesită cunoștințe tehnice, permițând oricui să filtreze și să compare aceste date pe mai multe criterii.

2.2 Cerințe non-funcționale

  • Responsive design — breakpoint-uri la 500px, 700px, 900px
  • Securitate — SQL Injection prevenit prin PDO::prepare(); XSS prevenit prin folosirea textContent în JavaScript și returnarea de JSON pur din API
  • Arhitectură bazată pe servicii Web — comunicare client-server exclusiv prin AJAX (fetch())
  • Fără framework-uri — JavaScript vanilla pe client, PHP pur pe server
  • Licență liberă — MIT

2.3 Fluxul utilizatorului

  1. Selectează județele dorite din lista multi-select
  2. Setează intervalul temporal cu selectoare de tip month
  3. Apasă „Aplică filtre" — datele se actualizează asincron
  4. Navighează între tab-uri (Rată, Urban/Rural, Vârste, Educație)
  5. Consultă harta coroplețică și statisticile sumare
  6. Exportă datele sau graficele în formatul dorit

4. Arhitectura aplicației

Aplicația urmează arhitectura clasică client–server, cu comunicare exclusiv prin apeluri AJAX asincrone. Nu se folosesc framework-uri pe nicio parte; tot codul rulează pe PHP cu PDO și SQLite pe server, respectiv JavaScript vanilla pe client.

5.2 Hartă coroplețică (Leaflet.js)

Fiecare județ este reprezentat printr-un circleMarker poziționat la coordonatele geografice ale județului. Culoarea fiecărui marker este calculată dinamic pe baza ratei șomajului pentru luna curentă, folosind un gradient de la galben (rată mică) la roșu (rată mare). Tile-urile provin de la OpenStreetMap via Leaflet.js.

5.3 Export

  • CSV / JSON — generate server-side de export.php, cu filtrele active aplicate
  • SVG — generat client-side prin capturarea canvas-urilor Chart.js ale tab-ului activ, înglobate într-un fișier SVG
  • PDF — prin window.print() cu reguli @media print în CSS

5.4 Securitate

  • SQL Injection — toate query-urile folosesc PDO::prepare() cu parametri legați; nicio valoare din input nu este interpolată direct în SQL
  • XSS — API-urile returnează exclusiv JSON; în JavaScript, datele dinamice sunt inserate în DOM prin textContent (nu innerHTML), eliminând posibilitatea de injectare HTML
  • Autentificare admin — sesiune PHP verificată pe fiecare endpoint administrativ; răspuns 403 la acces neautorizat

6. Concluzii

Aplicația UnD acoperă cerințele proiectului: 17 luni de date, filtrare multi-criterială, 7 tipuri de grafice, hartă coroplețică, export în 4 formate și modul de administrare complet. Implementarea fără framework-uri a necesitat înțelegerea mecanismelor fundamentale: HTTP, AJAX, REST, SQL, sesiuni PHP și manipularea DOM-ului din JavaScript.
